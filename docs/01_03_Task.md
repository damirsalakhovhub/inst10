Task Management

Acceptance criteria
- Юзер может создать задачу в своем проекте с title, description, status, датами.
- Юзер может редактировать свою задачу.
- Черновик (is_draft=true) виден только создателю, не отображается в общем списке.
- Опубликованная задача (is_draft=false) видна всем участникам проекта.

Jobs to be done

1. Task Model
   ◽️ Миграция: project_id, user_id, title, description, status, priority, start_date, due_date, is_draft, created_at, updated_at
   ◽️ Связь belongs_to project
   ◽️ Связь belongs_to user (автор)
   ◽️ Валидация presence для title и project_id
   ◽️ Enum для status: to_do, in_progress, done
   ⏸️ Enum для priority: low, medium, high

2. CRUD Operations
   ◽️ Создание задачи с автором (current_user)
   ◽️ Редактирование всех полей задачи
   ⏸️ Удаление задачи
   ◽️ Список задач проекта (только опубликованные для общего списка)

3. Draft Logic
   ◽️ По умолчанию задача создаётся как черновик (is_draft=true)
   ◽️ Scope для фильтрации черновиков и опубликованных задач
   ◽️ Метод publish для перевода черновика в опубликованную задачу
   ◽️ UI кнопка "Publish" на черновике

4. Authorization
   ◽️ Pundit политика: только автор может редактировать/удалять задачу
   ◽️ Только автор видит свой черновик
   ◽️ Опубликованные задачи видят все участники проекта (пока owner)

5. Routes
   ◽️ GET /projects/:project_id/tasks - список задач проекта
   ◽️ GET /tasks/:id - карточка задачи
   ◽️ POST /projects/:project_id/tasks - создание задачи
   ◽️ PATCH /tasks/:id - редактирование
   ◽️ DELETE /tasks/:id - удаление
   ◽️ POST /tasks/:id/publish - публикация черновика

6. UI
   ◽️ Список задач проекта с фильтром по статусам
   ◽️ Форма создания/редактирования задачи
   ◽️ Карточка задачи с полным набором полей
   ◽️ Индикатор черновика на карточке
   ◽️ Кнопка "Publish" для черновиков
   ⏸️ Кнопки Edit/Delete на карточке задачи

7. Validation & UX
   ◽️ title не может быть пустым
   ⏸️ start_date не может быть позже due_date
   ◽️ priority и status имеют дефолтные значения
   ◽️ Сообщения об ошибках валидации на формах
