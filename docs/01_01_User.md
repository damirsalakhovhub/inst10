User Management

Acceptance criteria
- Новый юзер может зарегистрироваться через email/пароль.
- Юзер может войти и выйти из системы.
- Неавторизованный юзер не видит приватных данных.

Jobs to be done

1. Authentication
   ✅ Email/password регистрация через Devise (реализовано на landing)
   ⏸️ Подтверждение email опционально (можно отключить для MVP)
   ✅ Login/logout flow (реализовано на landing + header)
   ✅ Remember me функционал (реализовано на форме входа)

2. Password Management
   ✅ Сброс пароля через email (есть в Devise, ссылка добавлена в UI на landing странице)
   ✅ Минимальные требования к паролю (8+ символов) (настроено в Devise, протестировано)
   ✅ Защита от brute-force (rate limiting) (rack-attack настроен)

3. User Model
   ✅ Миграция с полями: email, encrypted_password, reset_password_token, timestamps (Devise)
   ✅ Валидация email (формат, уникальность) (Devise, протестировано)
   ✅ Хранение паролей через bcrypt (встроено в Devise)

4. Authorization Basics
   ✅ Pundit политики для доступа к ресурсам (настроено, UserPolicy реализована)
   ◽️ Базовая политика: юзер видит только свои проекты и задачи (не реализовано, проектов/задач еще нет)
   ◽️ Редирект неавторизованного юзера на login (частично: authenticate_user! только в UsersController)

5. Security Basics
   ✅ CSRF токены на всех формах (Rails автоматически)
   ✅ Защита от session fixation (Devise автоматически)
   ✅ HTTP-only cookies для сессий (Devise автоматически)
   ✅ Rate limiting на login/signup (rack-attack настроен)

6. UI
   ✅ Форма регистрации (реализовано на landing)
   ✅ Форма входа (реализовано на landing)
   ✅ Форма сброса пароля (есть в Devise, ссылка "Forgot your password?" добавлена на landing)
   ✅ Remember me функционал (реализовано на форме входа, протестировано)
   ✅ Автозаполнение email (добавлено autocomplete="email")
   ⏸️ Базовый профиль юзера (опционально для MVP)
