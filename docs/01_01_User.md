User Management

Acceptance criteria
- Новый юзер может зарегистрироваться через email/пароль.
- Юзер может войти и выйти из системы.
- Неавторизованный юзер не видит приватных данных.

Jobs to be done

1. Authentication
   ✅ Email/password регистрация через Devise
   ⏸️ Подтверждение email опционально (можно отключить для MVP)
   ✅ Login/logout flow
   ⏸️ Remember me функционал

2. Password Management
   ⏸️ Сброс пароля через email
   ✅ Минимальные требования к паролю (8+ символов)
   ⏸️ Защита от brute-force (rate limiting)

3. User Model
   ✅ Миграция с полями: email, encrypted_password, reset_password_token, timestamps
   ✅ Валидация email (формат, уникальность)
   ✅ Хранение паролей через bcrypt (встроено в Devise)

4. Authorization Basics
   ✅ Pundit политики для доступа к ресурсам
   ◽️ Базовая политика: юзер видит только свои проекты и задачи
   ◽️ Редирект неавторизованного юзера на login

5. Security Basics
   ✅ CSRF токены на всех формах
   ⏸️ Защита от session fixation
   ⏸️ HTTP-only cookies для сессий
   ⏸️ Rate limiting на login/signup (rack-attack)

6. UI
   ✅ Форма регистрации
   ✅ Форма входа
   ⏸️ Форма сброса пароля
   ⏸️ Базовый профиль юзера (опционально для MVP)
