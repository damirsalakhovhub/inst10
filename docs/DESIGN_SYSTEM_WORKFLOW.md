Design System Workflow

В этом проекте мы используем дизайн-систему из другого проекта rails-template (https://github.com/damirsalakhovhub/rails-template).

Все компоненты, токены и стили хранятся там, а сюда подтягиваются через git subtree. Это значит что файлы дизайн-системы будут лежать в папке design_system и выглядеть как обычные файлы проекта. Когда нужен новый компонент, например селект, ты создаешь его прямо здесь в inst10, тестируешь и коммитишь. Потом переключаешься на rails-template, копируешь туда файлы компонента и коммитишь. Возвращаешься в inst10, обновляешь дизайн-систему командой git subtree pull, удаляешь локальный компонент из app/components и обновляешь импорты чтобы использовать версию из design_system. После этого компонент доступен везде где используется дизайн-система.

Работать можно в обычных feature-ветках, отдельная ветка для UI kit не нужна. При переключении между проектами и ветками ничего дополнительного делать не надо, потому что файлы дизайн-системы хранятся как обычные файлы в git. Если обновил что-то в rails-template и нужно подтянуть изменения в inst10, просто запускаешь git subtree pull с указанием пути design_system, репозитория и ветки. Все изменения появятся как обычный коммит, который можно проверить и закоммитить.
