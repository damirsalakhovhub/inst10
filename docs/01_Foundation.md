Фундамент проекта

Acceptance criteria
- Юзер может зарегистрироваться, войти, создать проект.
- Юзер может создать задачу в проекте, открыть карточку задачи, отредактировать её.
- Черновик (is_draft=true) не отображается в общем списке задач проекта до публикации.
- Только создатель черновика видит его в интерфейсе.

Цели
- Запустить скелет приложения так, чтобы дальше можно было быстро наращивать фичи без переделок.
- Зафиксировать минимальную доменную модель и базовые экраны.

Jobs to be done

1. User Management [01_01_User.md]
   ◽️ Регистрация и авторизация
   ◽️ Базовая валидация данных юзера
   ⏸️ Сброс пароля через email
   ◽️ Базовые политики доступа

2. Project Management [01_02_Project.md]
   ◽️ Миграция и модель Project
   ◽️ CRUD для проектов
   ◽️ Список проектов юзера
   ◽️ Политики доступа к проекту

3. Task Management [01_03_Task.md]
   ◽️ Миграция и модель Task
   ◽️ CRUD для задач
   ◽️ Список задач проекта
   ◽️ Карточка задачи
   ◽️ Черновики и публикация
   ◽️ Базовая валидация полей

4. UI Infrastructure
   ◽️ Базовая структура папок для компонентов
   ◽️ Общие layout и навигация
   ◽️ Базовые формы для Project и Task
   ◽️ Списки и карточки

5. Dev Environment
   ◽️ База данных подключена
   ◽️ Миграции работают
   ◽️ dev/test окружения поднимаются одной командой
   ◽️ Базовые тесты на критичные потоки
