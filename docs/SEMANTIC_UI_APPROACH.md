# Semantic UI: Подход к базовым компонентам

## Философия

**"Classes use syntax from natural languages"** — классы читаются как естественный язык (существительное/модификатор).

## Кнопки

### Размеры (Sizes)
Простые и понятные размеры:
- `mini` — самый маленький
- `tiny` — очень маленький
- `small` — маленький
- `medium` — средний (по умолчанию)
- `large` — большой
- `big` — очень большой
- `huge` — огромный
- `massive` — максимальный

Каждый размер задает пропорциональные изменения padding, font-size, border-radius.

### Типы (Types)
Базовые типы определяют стиль и назначение:
- `primary` — основная кнопка (обычно синяя)
- `secondary` — вторичная (обычно серая)
- `positive` — позитивное действие (зеленая)
- `negative` — негативное действие (красная)
- `basic` — минималистичная
- `inverted` — инвертированная

### Состояния (States)
- `disabled` — отключена
- `loading` — загрузка (с индикатором)
- `active` — активна
- `compact` — компактная (меньше padding)

### Композиция
Модификаторы комбинируются через пробел:
```html
<button class="ui button primary large">Submit</button>
<button class="ui button secondary small disabled">Cancel</button>
```

### Иконки
Иконки — отдельный компонент, легко добавляется:
```html
<button class="ui button">
  <i class="heart icon"></i>
  Like
</button>
```

## Принципы

1. **Предсказуемость** — одинаковые модификаторы дают одинаковый результат
2. **Композиция** — модификаторы независимы, можно комбинировать
3. **Семантика в именах** — названия отражают назначение (`primary`, `positive`)
4. **Минимальный набор** — только нужные варианты, без избыточности
